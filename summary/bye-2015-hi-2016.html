说不后悔的2015，说要加油的2016 2016/01/10

2015是非常特别永远不会忘记的一年，暂别学生时代，正式踏入成人世界。

分几块来总结吧：

  1. 1. 技术（或者说工作）
  2. 2. 兴趣
  3. 3. 软实力

在说工作之前，先飞快地总结一下 Master 时期。

记得高中的时候，数学老师永远在重复“预习，上课，复习”，这一点其实是到了 Master 的最后一个学期才尝试的，然而并没有什么用。学生能不能学好，作为我来说，老师够不够吸引才是最重要的，所以男神老师的课我学很好啊。

所以开始 Master 之前说“我要拿全A”的话真是说说而已啊- -，想到 dmc 还拿得是 B。然后第二学期的 CG 前半学期在教 SVG，后半学期在教 3D CG，结果期末考 SVG 部分差点得 0 分... 简直愧对我的 title。

手算到死的 Data Mining 是 A+，要感谢总是穿爱马仕的 Dr. Chen。虽然现在绝对各种忘光光，但是那些奇妙的挖掘方法已经存进了我的长期记忆，只要契机出现了，它们就会被唤醒。

很不想把念过的数据挖掘、图形学、RP还有数据结构算法……那么东西忘记掉，但是真的工作以后用到的地方少了很多，虽然坚信总有一天会用到，但是现在真的很虚。

技术/工作

项目很多，只说对我而言有里程碑意义的吧：

  1. 1.

    企鹅 FM 鬼节盗墓笔记活动页

    这是第一个主动要求承担的需求，毕竟是盗墓笔记看了800遍的哥嫂粉。虽然真正在做需求的时候内心一直在呐喊“这页面和《盗墓笔记》有毛关系啊”，但是因为很喜欢活动页的设计，还有页面上各种各样的效果，还是做得非常非常开心。最重要的是，点亮了一些重要的技能： animation、页面兼容、CSS3 3D。也踩到了一些坑：安卓上伪元素动画失效。具体的总结可以看这篇文章。

    念书的时候，我会收集业界前端团队的招聘信息，比如UED啦、奇舞团啦还有当年非常非常想要进的CDC，再有就有ISUX。自然也会看每个团队的技术文章。到有一天，我自己的文章被发布在这个平台上。那种心情，只有“为了让我的名字出现在借记卡上，我看了好多好多的书”可以形容。

    所以啊，这个项目对我来说，意义非凡。

  2. 2.

    企鹅FM 管理端

    这是一个 PC 项目，这是腾讯大老板的 PC 项目，这是用户量随随便便就很大的项目，所以要兼容到 IE7。虽然我严苛的导师大人差点（失去理智）要我兼容到 IE6，但在坚决表示“我不下模拟器”最终还是算了。

    很久没有接触 PC 端的项目了，不能用 flexbox 来布局，真 的 很 不 爽。回归到传统的 float、table还有多行文本垂直居中。期间还遇到了 IE7 下的几个绝对定位相关的坑，都放在了自己的 evernote 上了。

  3. 3.

    微云签到活动

    让我直视了 webview 和 X5 内核是要分别对待的事情。点亮了波纹动画的技能点。坑也是有踩，但是总结还是欠了一篇。另外，在微云这个项目中，接触很多图片相关的需求，关于图片的总结也是欠了一篇的。

    重构非常神奇，通过 CSS 属性的排列组合加上不同的切图方式，最后得到的页面效果会千奇百怪。即使页面长得一样，性能也会相差甚远。

  4. 4.

    企鹅FM 点歌台

    记得念书的时候，我妈常说“开窍”这件事情，我是个慢半拍。所以当导师大人一开始和我解释应该要如何考虑 HTML 的结构、要如何决定 class 如何命名还有不同的样式要怎么放……这些东西的时候，我也是迷迷糊糊，一边担心一边想着也许多做项目我就知道了。到了企鹅FM 点歌台的这个项目，好像终于找到了一点点感觉。

    again，欠一篇总结…

    怎么感觉这篇 post 写出来是在细数自己欠了债用的……挖坑给自己跳。

稍微总结一下这 part。

人在成长，也在丢失。

在还是动画小新手的时候，我真的有牢记动效原则，但后来好像随着页面越写越多，就渐渐忘记要坚守的原则了。希望这一点可以被拾起来。

可以明显地感觉到自己写页面的速度快了很多，但是移动端兼容却做得不够好。

也明显感觉到对产品的考虑比以前要全面了，一些边界问题会自然地想到。

2016 年的计划

就分工作和生活来说吧。

  1. 1.

    工作

    2015 年没有做好的地方：动画原则和兼容，今年要做好。这些是可以看得到的，视觉同学的调整情况、测试同学给我的 bug 单的数量都可以反馈。

    工作以后，对新技术的关注真的少了很多。从前不相信，现在真是不得不承认，总说着'T'型人才，T 字的翅膀都没展开呢，这就开始往下边挖了吗？这也是我要好好思考的问题。

    希望点亮 WebGL 的技能。我有图形学的梦想在，而且又是 CSS 感兴趣的人。WebGL 是这两者结合，希望对得起自己的“喜欢”。衡量的标杆就是，建一个模型出来，当年学 CG 建了一个垃圾桶和一个机器人。

    如果有用到 SVG 或者 canvas 的场景，一定要用上。

    要用 React 做一个应用，react.js 也好，react native 也要。没有用过 React，明明已经有了一台 Mac，却还没有尝试 React，这是我今年的一大遗憾吧。

    最后一个就是，希望可以修炼自己的 PS 技能。可以模仿图标。

    别的技术，就随缘了。
